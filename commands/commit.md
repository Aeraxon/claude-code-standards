Review the current changes and create a commit:

1. Run `git diff --staged` to see changes
2. Categorize the change type (feat/fix/docs/style/refactor/test/chore)
3. Create meaningful commit message following Conventional Commits:
   - type(scope): subject
   - body (optional, explain what and why)
   - footer (optional, reference issues)
4. Suggest the commit
5. Wait for approval before committing
